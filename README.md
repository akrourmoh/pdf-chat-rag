# PDF Chat (RAG)

A web application that lets users upload one or more PDF documents and ask
questions about their content in natural language. Answers are generated by a
Large Language Model (Google Gemini) and grounded strictly in the uploaded
documents using a Retrieval-Augmented Generation (RAG) pipeline.

---

## Overview

Large Language Models do not have access to a user's private documents and can
produce confident but incorrect answers when asked about content they were never
trained on. This project addresses that problem with **Retrieval-Augmented
Generation (RAG)**: instead of relying on the model's memory, the application
first retrieves the most relevant passages from the uploaded PDFs and then asks
the model to answer using only that context. The model is explicitly instructed
to respond with "I don't know based on the uploaded documents" when the answer is
not present, which reduces hallucinations.

## Features

- Upload and query multiple PDF files at once.
- Conversational chat interface with message history.
- Document-grounded answers (the model is constrained to the retrieved context).
- Fast semantic search over document content using a vector index.
- Lightweight, API-based embeddings that run comfortably on free hosting tiers.

## Architecture

The application implements a standard RAG pipeline. Document ingestion (steps 1-4)
runs once when the user clicks "Process PDFs"; retrieval and generation (steps 5-6)
run on every question.

| Step | Responsibility | Module |
|------|----------------|--------|
| 1. Load | Extract raw text from each PDF | `rag/loader.py` |
| 2. Chunk | Split text into overlapping segments | `rag/chunker.py` |
| 3. Embed | Convert text chunks into vectors via the Gemini API | `rag/embedder.py` |
| 4. Index | Store vectors in a FAISS index for fast search | `rag/vectorstore.py` |
| 5. Retrieve | Find the most relevant chunks for a question | `rag/vectorstore.py` |
| 6. Generate | Prompt Gemini with the question and retrieved context | `rag/chain.py` |
| UI | Orchestrate the pipeline and render the chat interface | `app.py` |

**Data flow:**
`PDF -> text -> chunks -> embeddings -> FAISS index -> (question) -> retrieved passages -> Gemini answer`

## Tech Stack

- **Language:** Python
- **Web framework / UI:** Streamlit
- **LLM:** Google Gemini (`gemini-2.5-flash`)
- **Embeddings:** Google Gemini (`gemini-embedding-001`)
- **Vector search:** FAISS
- **PDF parsing:** PyMuPDF
- **Text splitting:** LangChain `RecursiveCharacterTextSplitter`
- **Deployment:** Render

## Getting Started (Local)

### Prerequisites

- Python 3.10 or newer
- A Google Gemini API key ([Google AI Studio](https://aistudio.google.com/app/apikey))

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/akrourmoh/pdf-chat-rag.git
cd pdf-chat-rag

# 2. (Recommended) Create and activate a virtual environment
python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS / Linux
source .venv/bin/activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Configure your API key
#    Create a file named .env in the project root containing:
#    GOOGLE_API_KEY=your_key_here

# 5. Run the app
streamlit run app.py
```

The application will open in your browser at `http://localhost:8501`.

## Deployment

The project is configured for one-click deployment on [Render](https://render.com)
via the included `render.yaml` blueprint.

1. Push the repository to GitHub.
2. In Render, create a new **Web Service** from the repository. Render reads
   `render.yaml` automatically.
3. Add an environment variable `GOOGLE_API_KEY` with your Gemini API key.
4. Deploy. Render provides a public URL you can share.

## Configuration Notes

- The Gemini API key is read from the `GOOGLE_API_KEY` environment variable. It is
  never committed to source control (`.env` is listed in `.gitignore`).
- Embeddings are generated through the Gemini API rather than a local model, which
  keeps the memory footprint small enough to run on free hosting tiers.
